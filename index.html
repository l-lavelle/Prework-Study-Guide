HTML

  • Used to create the structure of a webpage
  • Made up of elements that work together to provide content and basic framework for all webpages
    • Elements perform unique roles to divide the content on the page and describe it to the browsers and other developers
    • Elements are composed of opening and closing tags
    • Attributes define additional information about elements
  • The head element contains information about the webpage.
    • Elements included in the head are not visible when a user looks at a webpage
    • Provides important information to browser, search engine, and developers
  • The body element represents the visible content shown to the user.
    • Broken down into header, main, footer elements creating different sections
    • Want to use Semantic HTML elements when possible

CSS

  • Use to define style rules for specific elements of HTML page and format them using wide range of style options
  • Types of CSS
    • Inline CSS: used to style elements directly in HTML file
    • Internal Style Sheet: contains CSS rules for the webpage in head section of HTML file
    • External Style Sheet: keep all CSS rules in separate file and link it to HTML file
  • CSS Rule: uses selector to define the element to which the rules will apply and properties and values to create declarations
  • Class attributes are used to add styles to certain specific elements instead of all elements
  • CSS Box Model
    • Every element in CSS is a box
    • Each box consists of margins, borders, padding, and content
    • Margin: defines how much space wanted around outside of element
    • Padding: defines how much space wanted around content inside an element

Git

  • Open source version control system which manages changes of documents and programs
  • Master Branch
    • Contains main working, tested, and ready to deploy code
  • Feature Branches
    • Copy of repo that feature deployment work takes place on
    • Makes it easier for multiple developers to work on particular features without disturbing main codebase
  • Github Issues
    • Keeps track of tasks and projects
  • Git Commands
    • git clone: brings repo that is hosted elsewhere into folder on local machine
    • git status: checks what branch we are currently on
    • git checkout -b branch-name: creates a new branch and switches to it
    • git Add -A: adds files to staging area
    • git commit -m "message": saves file in git
    • git pull origin master: download changes from remote repo to local machine
    • git push origin branch-name: upload git commits to remote repo

JavaScript

  • Adds interactivity to website
  • For js file to work have to link to html file
  • Control flow is the order in which a computer executes code in a script. Starts from top and exceutes in order to bottom
  • Conditional Formatting: used to perform different actions based on different conditions
  • For loop: executes a block of code a certain number of times
    • Contains: Starting point for loop, condition, and iteration
  • A variable is a named container that allows us to store data in our code
    • Declare variable using keyword var
  • Array: single variable that is used to hold group of data
    • Arrays can store different types of data
    • Use indexing to locate individual data in array
  • Function: set of instructions that rells the computer how to perform certain tasks
    • Must call function name in order to execute it
    • Helps with reusing code and control flow
